# Классификация типов связей в схеме кинотеатра

## 1. Один-ко-многим (One-to-Many)

**movie → oscar_awarded**  
Один фильм может получить несколько наград Оскар, но каждая награда принадлежит только одному фильму.  
При удалении фильма все связанные награды автоматически удаляются (ON DELETE CASCADE).

## 2. Многие-ко-многим (Many-to-Many)

**movie ↔ director (через movie_direction)**  
Один фильм может быть снят несколькими режиссерами, и один режиссер может снять несколько фильмов.  
При удалении фильма или режиссера автоматически удаляются соответствующие записи в связующей таблице.

**movie ↔ actors (через movie_cast)**  
Один фильм может содержать несколько актеров, и один актер может сниматься в нескольких фильмах.  
При удалении фильма или актера автоматически удаляются соответствующие записи о ролях.

## Поведение при удалении:

Во всех связях используется ON DELETE CASCADE, что означает:

- При удалении родительской записи (например, фильма) автоматически удаляются все связанные дочерние записи (награды,
  связи с режиссерами и актерами).
- Это обеспечивает целостность данных и предотвращает появление "висячих" ссылок.

## Особенности:

1. В таблице `movie_cast` составной первичный ключ (act_id, mov_id, role) позволяет одному актеру играть несколько ролей
   в одном фильме.
2. В таблице `actors` поле act_gender имеет ограничение CHECK для допустимых значений.
3. Все обязательные поля помечены как NOT NULL.
